Phased Migration Plan
5 phases · Estimated 6–8 weeks total · Can be executed incrementally
⚠ Current Problems
  • Advances scattered in flat monthly folders with no event context
  • Duplicate files named "(1)", "(2)" throughout (especially Advances)
  • Loose files at the root of JSP Master Shared Drive
  • IT, HR, Finance mixed with creative & event content
  • _BANDS_CORE lives orphaned with no parallel structure for IVC/Keynote talent
  • Archive events from 2017–2021 mixed with active 2025–2026 events
  • Inconsistent naming: some use full names, some codes, some both
  • Pitch decks have no version or client context in filename
✓ After Reorg
  • Every event folder is self-contained: contracts, advance, P&L, talent, marketing
  • Single Advances folder per event — no dupes, clear versioning
  • Talent hub with consistent subfolders per artist (bio, contracts, tech rider, media)
  • Ops, Finance, HR, IT each in their own top-level silos
  • Archive clearly separated so active work is never cluttered
  • Naming convention applied consistently across all files
  • Shared Drive root has zero loose files
1
Phase 1 · Week 1

Audit & Freeze

Days 1–5 · Low risk · No files moved
Before touching anything, get a full picture of what exists. This phase is observation only — nothing is deleted or moved.
  • Export gdrive.json and build a full inventory spreadsheet (file count per folder)
  • Flag all duplicate files (search for "(1)", "(2)" in names)
  • Identify the 20 most-used folders via Drive activity
  • Document all current top-level folders and their purpose
  • Communicate freeze period to team — no new folders created
  • Get team alignment on proposed new structure before any moves
2
Phase 2 · Week 2

Build New Skeleton

Days 6–10 · Zero risk · Structure only
Create the new folder structure without moving any existing files. Test it feels right before committing.
  • Create top-level folders: 📁 EVENTS, 📁 TALENT, 📁 OPERATIONS, 📁 BUSINESS DEV, 📁 ARCHIVE
  • Build EVENTS → by Year → by Month substructure
  • Create TALENT hub with one test artist subfolder as a template
  • Create OPERATIONS subfolders: Finance, HR, IT, Legal, Playbooks
  • Create BUSINESS DEV subfolders: Proposals, RFPs, Pitches, Marketing
  • Share skeleton with team for feedback before migration begins
3
Phase 3 · Weeks 3–4

Migrate Active Events

2025–2026 events only · Highest priority
Move all current and upcoming events into the new structure. This is the highest-impact phase — start here to feel the benefit immediately.
  • Move 2026 event folders into EVENTS → 2026 → [Month]
  • Move 2025 event folders into EVENTS → 2025 → [Month]
  • Within each event folder: standardize subfolders to: Client, Talent, Advance, P&L, Marketing, Venue
  • Move loose Advance docs (currently in flat monthly folders) into their correct event folder
  • Delete confirmed duplicate Advance files after verifying final version
  • Move all FEC / SNG Shared Drive content into TALENT hub
4
Phase 4 · Week 5

Migrate Ops & Talent

Internal infrastructure reorganization
Consolidate all operational, HR, finance, and talent roster content into the new silos.
  • Move Finance Meeting notes into OPERATIONS → Finance → by Year
  • Move HR folder content into OPERATIONS → HR
  • Move IT files (Airtable, Copper, Domains, Tech Stack) into OPERATIONS → IT
  • Move Playbook and Process Docs into OPERATIONS → Playbooks
  • Move _BANDS_CORE into TALENT → Core Bands
  • Move IVC Strategy folder into TALENT → Keynote Speakers
  • Move Philanthropy docs into OPERATIONS → Philanthropy
  • Move Staff Meeting notes into OPERATIONS → Internal Meetings
5
Phase 5 · Weeks 6–8

Archive & Polish

Historical events · Cleanup · Documentation
Move historical events to archive, clean up the root, apply naming conventions, and document the new system for the team.
  • Move all pre-2024 event folders into ARCHIVE → [Year]
  • Move Armed Forces Entertainment tour docs into ARCHIVE → 2017
  • Clean root of JSP Master Shared Drive — zero loose files should remain
  • Apply naming convention audit to all 2025–2026 event files
  • Write Drive SOP doc (where things live, naming rules, how to create new event folders)
  • Train team on new structure via Loom or staff meeting walkthrough
  • Set up quarterly Drive audit reminder to prevent re-accumulation of clutter

Naming Convention Guide
Consistent naming = searchability · Apply to all active files going forward
📁 Event Folder Name
[CODE] | [Artist/Talent] | [Client/Event Name] | [Venue] | [City]
✅ JSP100825 | Colin Jost & Michael Che | Predict | Recorded Future | NYC
[CODE] = Band abbreviation + MMDDYY (e.g., WB5, JSP, POP, NOT, TCB, BBX)
Pipe separators keep folder names scannable across all screen sizes
📄 File Naming: Contracts
[Type] ([Party]) - [CODE] - [Artist] - [City]
✅ Executed Agreement (Client) - JSP100825 - Colin Jost - NYC.pdf
✅ Contract Agreement (Talent) - WB5091325 - Wonder Bread 5 - Austin.gdoc
📄 File Naming: Advances
[CODE] - ADVANCE - [Artist] (FINAL)
✅ WB5091325 - ADVANCE - Wonder Bread 5 (FINAL).gdoc
❌ WB5091325 - ADVANCE (1).gdoc · WB5091325 - ADVANCE (2).gdoc
Rule: Only ONE advance doc per event. Use Google Doc version history for revisions — no duplicate files.
📄 File Naming: P&L
[CODE] - PRV - [City][Artist] Profit and Loss
✅ JSP100825-PRV-NYCColinJostMichaelChe Profit and Loss.pdf
📁 Talent Artist Subfolder (in TALENT hub)
[ARTIST NAME] (all caps, consistent)
✅ WONDER BREAD 5 / DJ JAZZY JEFF / LACE LARRABEE
SUBFOLDERS INSIDE EACH ARTIST:
📁 Contracts
All talent agreements
📁 Tech Rider
Input list, stage plot
📁 Media
Photos, logos, EPK
📁 Financial
W-9, invoices, payments
⚠ THINGS TO STOP DOING
  • Saving duplicate Advance files as "(1)", "(2)" — use version history instead
  • Storing files at the root of the Shared Drive
  • Creating event folders without the standard 6-subfolder structure
  • Naming Pitch Decks "Pitch.gslides" or "Pitch (3).gslides" — include client name and year
  • Mixing archived events with active events in the same folder
  • Creating personal/individual folders inside the shared drive (use My Drive for personal notes)
